        # 本地重新拉取jar包的步骤

        1. 先删除本地maven仓库中该jar包所在文件夹
        2. 到任意一个有pom.xml文件的文件夹下，打开cmd
        3. 执行： mvn dependency:get -Dartifact=groupId:artifactId:version

        # 手动上传snapshot-jar包到私服Nexus的方法

        由于Nexus私服的Release仓库不允许上传snapshot版本包，且SNAPSHOT仓库没有对应的Web上传界面。 故只能通过maven命令实现上传。

        mvn deploy:deploy-file -DgroupId=... -DartifactId=... -Dversion=1.0-SNAPSHOT -Dpackaging=jar -Dfile=F:\jar\xxx-1.0-SNAPSHOT.jar -Durl=http://127.0.0.1/nexus/content/repositories/snapshots/ -DrepositoryId=snapshot
        • 参数解释：
          • -Dfile：要上传的jar包绝对路径
          • -Durl：私服SNAPSHOT仓库的地址
          • -DrepositoryId：本地maven配置文件settings.xml中server的id。比如
          // 相当于是要使用这个用户来登录才能上传，故此用户还需要有上传权限
<server>
  <id>releases</id>
  <username>admin</username>
  <password>123456</password>
</server>
<server>
  <id>snapshot</id>
  <username>admin</username>
  <password>123456</password>
</server>

        # cmd下报找不到mvn命令的解决方法

        将maven的bin目录全路径配置到环境变量Path中即可
